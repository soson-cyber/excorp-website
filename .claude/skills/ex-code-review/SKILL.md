# EX 코드 리뷰 (품질·보안)

커밋/병합 직전, **diff를 사람 리뷰어처럼** 읽고 옳음·안전·유지보수성을 판단한다. 빌드를 돌리지 않는다(그건 ex-qa-verify). 무관한 리팩터를 강요하지 않는다.

## 범위 설정
- `cd /Users/ex_ceo/Documents/Claude/Projects/excorp-website && git diff`(또는 변경 파일 목록)로 바뀐 부분과 영향 범위만 본다.
- 큰 diff는 모듈 단위로 나눠 보고, 못 본 범위를 명시한다.

## 보안 체크 (웹앱 — 최우선)
- **API Route(`app/api/*`)·서버 액션:** 입력 검증/스키마, 인젝션(SQL·명령·경로 traversal), SSRF, 인증·인가 누락, 과한 응답 노출.
- **시크릿:** 키·토큰·`.env` 값이 코드·로그·클라이언트 번들에 노출되지 않는가. `NEXT_PUBLIC_` 접두사 오용(서버 전용 값을 클라에 노출) 금지.
- **XSS:** `dangerouslySetInnerHTML`, Portable Text/마크다운 렌더러, 사용자 입력 반영 지점의 이스케이프.
- **개인정보:** 폼·쿼리스트링으로 개인정보를 URL에 싣지 않는가. 외부로 사용자 데이터를 흘리지 않는가.

**OWASP Top 10 빠른 대조** (변경에 해당하는 항목만): 접근 통제 실패(미인증 라우트) · 암호 실패(시크릿 평문·약한 해시) · 인젝션(SQL/명령/경로) · 안전하지 않은 설계(검증 우회 가능 흐름) · 보안 설정 오류(헤더·CORS·디버그 노출) · 취약·구식 의존성(새 패키지·버전) · 인증 실패(세션·토큰 처리) · 무결성 실패(검증 없는 외부 스크립트/CDN) · 로깅 누락 또는 민감정보 로깅 · SSRF(서버가 외부 URL 페치). 변경이 mutation(POST/PUT/DELETE)·외부 요청·인증을 건드리면 특히 집중한다. (도입 의존성은 supply-chain 관점에서 신뢰성·유지보수 상태도 본다.)

## 정확성
경계 조건, async/예외 처리 누락, React 19 함정(이펙트 내 동기 setState → rAF, `key` 누락, 의존성 배열, 조건부 훅), 타입 우회(`any`·`as` 단언 남용), 죽은 분기.

## 유지보수성·정합
중복/과한 추상화/죽은 코드, 네이밍, DESIGN.md 토큰·키트 클래스 일탈, 접근성 회귀(대비·포커스·탭타깃·reduced-motion), 정직성 규칙(카피·라벨 날조·리셀러/총판 정확).

## 심각도 태깅
- 🔴 **Critical** — 보안 취약·데이터 손실·빌드/배포 차단
- 🟠 **Major** — 명백한 버그·접근성 위반·정직성 위반
- 🟡 **Minor** — 품질·일관성·중복
- 🟢 **Nit** — 취향(여기에만)

## 출력
심각도별 목록 — 각 항목 `파일:라인 — 무엇 — 왜 — 제안`. 맨 위에 **차단 여부(Critical/Major 존재 시 "병합 보류 권고")**를 명시. 직접 수정하지 않고 담당 에이전트(주로 frontend-builder)에 반환. 빌드·렌더 재확인이 필요하면 qa-verifier에 위임. 배포는 사용자가 Vercel에서 수행하므로, 배포 전 차단 이슈는 리더에 명확히 보고한다.
