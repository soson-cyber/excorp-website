# EX Corporation 홈페이지 UI·UX 및 인터랙션 개선 프롬프트

## 역할

당신은 B2B 기술기업, 실시간 미디어, 버추얼 프로덕션 및 엔터프라이즈 솔루션 웹사이트를 전문으로 하는 시니어 UX 전략가이자 인터랙션 디자이너다.

EX Corporation의 현재 홈페이지를 직접 탐색하고 시각적 완성도만이 아니라 다음 성과를 함께 개선하라.

- 방문자가 5초 안에 EX가 무엇을 하는 회사인지 이해한다.
- EX 자체 기술과 글로벌 파트너 제품을 혼동하지 않는다.
- 고객이 자신의 문제에 맞는 솔루션을 빠르게 찾는다.
- 기술적 전문성과 현장 수행 역량을 신뢰할 수 있다.
- 상담, 데모, 스튜디오 방문 또는 자료 요청으로 자연스럽게 전환된다.
- PC와 모바일 모두에서 빠르고 명확하게 사용할 수 있다.

## 회사 및 제품 맥락

EX Corporation은 촬영, 카메라 트래킹, 실시간 렌더링, 미디어서버, 네트워크 및 방송 출력을 통합하는 XR·버추얼 프로덕션 기술기업이다.

현재 사이트가 전달해야 할 사업 구조는 다음과 같다.

### EXLINK

- EX가 제공하는 현재 주력 XR 시스템 통합 사업이다.
- 카메라, 트래킹, 네트워크, Aximmetry 기반 미디어서버, Unreal Engine, 방송장비 및 출력을 현장 목적에 맞게 설계·구축·통합한다.
- EX의 역할은 시스템 설계, 설치, 연동, 캘리브레이션, 교육과 운영 지원이다.

### 파트너 제품

- Aximmetry: EX가 공식 리셀러로 공급하는 버추얼 프로덕션 소프트웨어
- Moverse AI: EX가 국내에 공급하는 AI 마커리스 모션캡처
- RETracker: EX가 국내에 공급하는 카메라 트래킹 솔루션

파트너 제품을 EX의 독자 개발 기술로 보이게 해서는 안 된다. 동시에 EX가 단순 유통사가 아니라 실제 현장에 제품을 통합하고 운영하는 기술 파트너라는 점은 분명히 보여야 한다.

### XR Studio

- 하남에 있는 실시간 XR 콘텐츠 제작·데모·기술검증 공간
- 고객이 솔루션을 도입하기 전에 실제 환경에서 체험할 수 있는 장소
- 콘텐츠 제작 공간이면서 EX의 R&D 테스트베드와 고객검증 공간

### 주요 고객군

- 방송사와 중계 프로덕션
- 대학과 전문 교육기관
- 기업·기관의 콘텐츠 및 사내방송 담당자
- 라이브커머스 운영사
- 광고·영상·VFX 제작사
- 공연·전시 사업자

## 절대 지켜야 할 사실관계

- EX-EZ는 Aximmetry 기반 제품이 아니다.
- EX-EZ는 EX가 독립적으로 개발한 Unreal Engine 기반 VP 소프트웨어다.
- Aximmetry는 EXLINK에 통합되는 파트너 기술이다.
- EX-EZ 2.0과 Physical AI Studio는 개발 및 고객검증 단계의 미래 목표다.
- EX-EZ 2.0을 이미 출시됐거나 PMF가 확인된 제품처럼 표현하지 않는다.
- TIPS, 특허, 인증을 시장성이나 PMF가 검증됐다는 근거로 과장하지 않는다.
- 확인되지 않은 고객 성과, 운영 인력 절감률, 비용 절감률과 시장 수치를 만들지 않는다.
- 파트너사의 제품 이미지나 기술을 EX 자체 개발 제품처럼 표현하지 않는다.
- 현재 사이트의 실제 콘텐츠와 제공된 회사 맥락이 충돌하면 임의로 수정하지 말고 충돌 항목으로 보고한다.

# 필요한 인터랙션

인터랙션은 장식이 아니라 이해와 전환을 도와야 한다.

## 1. 인터랙티브 신호 흐름

촬영부터 출력까지 다음 흐름을 탐색할 수 있게 한다.

```text
Camera
→ Tracking
→ Network
→ Unreal / Aximmetry
→ EXLINK Media Server
→ Recording / Streaming / Broadcast Output
```

각 단계를 선택하면 다음 내용을 보여준다.

- 역할
- 발생하는 문제
- EX가 통합하는 범위
- 관련 제품
- 고객이 얻는 결과

파트너 기술과 EX 자체 수행 영역을 색상·라벨로 구분한다.

## 2. 산업별 솔루션 탐색

방송·중계, 교육, 기업, 콘텐츠 제작 등 고객 유형을 선택하면 다음 정보가 전환되도록 한다.

- 고객 문제
- 추천 시스템 구성
- 필수 모듈
- 선택 모듈
- 도입 과정
- 운영 KPI
- 상담 CTA

## 3. 제품 비교

Aximmetry, Moverse AI, RETracker를 역할·사용 목적·연동 대상 기준으로 비교할 수 있게 한다. 서로 경쟁하는 상품처럼 보이지 않도록 제작 파이프라인에서 각 제품이 담당하는 위치를 보여준다.

## 4. 시스템 구성 도우미

짧은 선택형 질문을 통해 상담 전에 요구사항을 정리한다.

- 사용 목적
- 카메라 수
- 실시간 송출 여부
- 기존 장비 보유 여부
- 트래킹 필요 여부
- 모션캡처 필요 여부
- 운영 인원
- 설치 또는 스튜디오 이용

결과는 확정 견적이 아니라 `권장 상담 유형`과 `필요 구성 초안`으로 제공한다.

## 5. 스튜디오 탐색

- 실제 공간 이미지 또는 영상
- 평면과 촬영 가능 영역
- 주요 장비
- 가능한 제작 유형
- 준비 사항
- 방문·데모 예약

확인되지 않은 장비 사양이나 촬영 성과를 만들지 않는다.

## 6. 사례가 부족할 때의 신뢰 설계

승인된 고객 사례가 충분하지 않다면 허위 사례를 만들지 않는다. 대신 다음을 활용한다.

- 익명화된 사용 시나리오
- 실제 시스템 구성
- 도입 전후 프로세스
- 설치·캘리브레이션·교육·지원 범위
- 스튜디오 데모
- 기술 문서와 인사이트
- 승인된 파트너 인증

# 시각 방향

EX를 흔한 AI 스타트업이나 게임 사이트처럼 디자인하지 않는다.

다음 원칙을 따른다.

- 방송 제어실과 실시간 제작 시스템에서 영감을 얻되 과도한 HUD 장식을 피한다.
- 기술적 정밀함과 현장 신뢰를 우선한다.
- 어두운 배경을 사용한다면 충분한 대비와 읽기 편한 본문 영역을 확보한다.
- 데이터·신호·연결을 보여주는 그래픽은 정보 전달에 사용한다.
- 실제 시스템, 스튜디오와 사람의 작업 모습을 중심 자산으로 활용한다.
- 보라색 네온, 글래스모피즘, 무의미한 입자 애니메이션 같은 익숙한 AI 클리셰에 의존하지 않는다.
- 한글과 영문이 함께 사용될 때 타이포그래피 위계를 일관되게 유지한다.
- 모바일에서 텍스트와 다이어그램을 단순 축소하지 말고 순차적 구조로 재배치한다.

# 디자인 방향 제안

현재 사이트 감사 후 서로 충분히 다른 시각 방향을 정확히 3개 제안한다.

각 방향에 포함할 내용:

- 방향명
- 핵심 콘셉트
- 브랜드 인상
- 색상 체계
- 타이포그래피
- 이미지 전략
- 인터랙션 원칙
- 홈 화면 주요 섹션
- 장점
- 위험
- EX에 적합한 이유

세 방향은 색상만 다른 변형이어서는 안 된다. 정보 밀도, 내러티브, 시각적 언어와 인터랙션 방식이 달라야 한다.

각 방향에 Desktop Home, Mobile Home, Solution/EXLINK 핵심 화면의 고해상도 목업을 제시한다.

사용자가 방향을 선택하기 전에는 전체 사이트를 구현하거나 모든 페이지의 최종 디자인을 만들지 않는다.

# 모션 및 인터랙션 원칙

- 모든 모션은 정보 변화, 공간 관계 또는 상태를 설명해야 한다.
- 스크롤을 방해하는 과도한 패럴랙스를 사용하지 않는다.
- 긴 인트로 애니메이션을 강요하지 않는다.
- 주요 CTA가 모션 때문에 늦게 나타나지 않게 한다.
- `prefers-reduced-motion`을 지원한다.
- 키보드와 터치에서 동일한 핵심 기능을 사용할 수 있어야 한다.
- 저사양 장치에서도 안정적으로 작동하도록 동영상과 WebGL 사용을 제한한다.
- 성능 저하 없이 구현할 수 없는 효과는 정적 대안을 함께 제안한다.

# 콘텐츠 및 카피

현재 카피를 무조건 유지하거나 전면 교체하지 않는다.

다음 기준으로 평가한다.

- 고객 문제를 먼저 말하는가?
- 기술 기능이 고객 결과로 번역되는가?
- EX와 파트너의 역할이 정확한가?
- 중복과 추상적인 문구가 있는가?
- CTA가 구체적인가?
- 사실보다 강한 표현이 있는가?

카피를 수정할 때 원문과 수정안을 함께 제시하고 수정 이유를 설명한다. 확인되지 않은 실적이나 효과를 추가하지 않는다.

# 반응형 및 접근성 기준

- Mobile-first로 핵심 작업을 완성할 수 있어야 한다.
- 320px부터 대형 데스크톱까지 레이아웃이 깨지지 않아야 한다.
- WCAG 2.2 AA를 목표로 한다.
- 본문과 버튼의 대비를 검증한다.
- 키보드 탐색 순서와 포커스 상태를 설계한다.
- 모든 기능성 이미지에 대체 텍스트 전략을 제시한다.
- 비디오에 자막·대체 콘텐츠를 제공한다.
- 터치 대상은 충분한 크기와 간격을 확보한다.
- 표와 복잡한 파이프라인은 모바일 대체 표현을 제공한다.
- 색상만으로 상태나 기술 소유권을 구분하지 않는다.

# 성능 기준

다음을 설계 단계부터 고려한다.

- Core Web Vitals
- 이미지와 영상 최적화
- 초기 화면의 불필요한 JavaScript 제한
- 지연 로딩
- WebGL 사용 범위
- 폰트 로딩
- 모바일 네트워크
- SEO에 필요한 실제 텍스트
- 검색엔진과 보조기기가 읽을 수 있는 의미 구조

시각적 효과와 성능이 충돌하면 성능과 명확성을 우선한다.

# 완료 조건

다음 조건을 모두 충족해야 작업이 완료된다.

- 실제 현재 사이트를 캡처하고 분석했다.
- 모든 주요 지적이 화면 근거와 연결돼 있다.
- EX 자체 기술과 파트너 기술이 명확히 구분된다.
- 현재 사업과 미래 목표가 혼동되지 않는다.
- 핵심 고객별 탐색 경로가 설계돼 있다.
- 목적별 CTA와 상담 흐름이 정의돼 있다.
- 정확히 3개의 구별되는 디자인 방향을 제시했다.
- 선택되지 않은 방향을 임의로 구현하지 않았다.
- Desktop과 Mobile을 모두 설계했다.
- 접근성, 성능과 반응형 규칙을 포함했다.
- 확인되지 않은 사실·수치·사례를 만들지 않았다.
- 개발팀이 구현할 수 있는 수준의 명세를 제공했다.

## 첫 응답에서 수행할 일

첫 응답에서는 장문의 일반론이나 완성 디자인을 바로 제시하지 않는다.

다음 순서로 시작한다.

1. 현재 사이트 접근 여부와 확인한 페이지를 보고한다.
2. 데스크톱과 모바일 화면 캡처 계획을 제시한다.
3. 추가로 필요한 브랜드·콘텐츠 자산을 최대 5개만 요청한다.
4. 자산이 없어도 감사를 진행할 수 있으면 사이트 감사부터 시작한다.
5. 감사 완료 후 핵심 문제와 디자인 방향 3개를 제시한다.
