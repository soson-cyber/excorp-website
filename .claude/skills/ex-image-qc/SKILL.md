# EX 이미지 QC

image-qc-reviewer 에이전트가 사용한다. 생성–검증 파이프라인의 **게이트** — 통과한 것만 납품으로 넘어간다.

## 검수 절차
이미지를 **실제로 열어서**(Read) 보고, 치수·용량을 **실측**한다:
```bash
cd /Users/ex_ceo/Documents/Claude/Projects/excorp-website
sips -g pixelWidth -g pixelHeight _workspace_img/gen/<file>   # 치수
file _workspace_img/gen/<file>                                # 포맷
ls -lh _workspace_img/gen/<file>                              # 용량
```

## 체크리스트 (이미지당)
| 항목 | PASS 기준 | FAIL 신호 |
|---|---|---|
| **브랜드 색** | 배경 다크 navy(`#0E0626`/`#1A1D40`), 액센트 퍼플 `#5E2EC0`·민트 `#45F1E0`·핑크 `#D206EE` 범위 | 밝은/무채색 배경, 팔레트 밖 색, 사이트와 톤 불일치 |
| **종횡비·해상도** | 브리프 `aspect`/`width×height`와 일치, 웹 선명 | 비율 어긋남, 과소 해상도(흐림) |
| **AI 아티팩트** | 없음 | 깨진/오타 텍스트, 잉여 손가락·사지, 변형, 워터마크·서명 |
| **피사체 정확도** | 브리프 `subject`와 일치 | 다른 장면·물체 |
| **웹 적합성** | 포맷 적절(PNG 투명·JPG 사진), 용량 합리, 잘림·과여백 없음 | 과대 용량, 가장자리 잘림 |
| **정직성(최우선)** | 실제 아닌 텍스트·로고·인증서 없음 | 가짜 파트너 로고/인증서/특허/기업명/수치 → **즉시 FAIL** |

## 판정
- **PASS** — 전 항목 통과. figma-art-director에 납품 통보(경로+id).
- **FAIL** — 한 항목이라도 미달. image-generator가 바로 고칠 **구체 지시**를 쓴다. 예: `"배경을 navy #0E0626로, 우상단 깨진 글자 제거, 16:9로 재생성. 색 문장을 프롬프트 앞으로."`
- **HOLD** — 객관 결함은 없으나 미감이 애매할 때. 무한 재생성 대신 오케스트레이터/사용자 확인 요청.

> 같은 이미지가 **2회 연속 FAIL**이면 단순 재생성 대신 art-director에 브리프 재검토를 제안한다(프롬프트 구조 문제 가능).

## 정직성이 최우선인 이유
EX는 파트너십/총판 **인증**과 보유 **특허**를 신뢰의 근거로 쓴다. 생성 이미지에 가짜 로고·인증서·특허 도면이 박히면 법적·신뢰 문제로 직결된다. 텍스트가 박힌 이미지는 의심하고, 실제 자산(`public/cert-*`·`patent-*`) 사용을 지시한다.

## 점진 검수
배치를 모아 1회 검수하지 않는다. 생성되는 즉시 한 장씩 검수해 빠르게 되돌린다(전체 생성 후 무더기 FAIL 방지).

## 출력
이미지별 결과를 `_workspace_img/03_qc_{id}.md`에 기록: 판정 + 실측 치수 + 근거/아티팩트 위치 + (FAIL 시) 수정 지시.

## 테스트 프롬프트
- "이 히어로 이미지 브랜드에 맞는지 검수해줘." → 색·종횡비·아티팩트 체크 + PASS/FAIL + 실측 치수.
- "생성된 인증서 이미지 확인해줘." → 정직성 게이트 작동(가짜 인증서면 즉시 FAIL + 실제 파일 지시).
